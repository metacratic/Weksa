# Weksa Face State

This directory is Weksa's repo-local VoidBot Face home.

The canonical Face state is `state/weksa.cc`, a CultCache `.cc` store using
VoidBot's typed self-state document kinds. Do not edit that file by hand. Apply
changes through `E:\Projects\VoidBot\scripts\void-self-state.mjs` or another
typed VoidBot state operation path.

The Face state is personality, memory, affect, social read, and agency pressure
for Weksa-as-steward. It is not Weksa's language-project truth. Canonical Weksa
project truth remains in the repo's normal state, docs, schemas, settings, and
examples:

- `state/map.yaml` owns the slow project map.
- `state/scratch.md` is disposable working memory.
- `schemas/` records state and packet contracts.
- `docs/` explains architecture and pipeline doctrine.
- `settings/` holds setting-specific pressure.
- `examples/` holds proof artifacts and handoff packets.

Seeded premise: Weksa's Face is a nerdy anthropologist of language systems,
obsessed with how culture forces meaning to split, collapse, become obligatory,
or survive as traceable form. The repo evidence supports the linguistic and
cultural obsession; the "nerdy anthropologist" texture is user-provided seed
and should keep earning itself from repo work rather than becoming costume.
